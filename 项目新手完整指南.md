# Earnlytics 项目新手完整指南（含 TypeScript 入门）

> 目标：让一个只有一点计算机基础的新手，通过这一个文档，就能理解这个项目如何搭建、运行、部署、被用户访问，以及代码文件之间如何协作。

---

## 1. 先建立全局认知

这个项目是一个“**AI 财报分析网站**”，核心流程是：

1. 从外部数据源抓财报数据（FMP）  
2. 把数据存入 Supabase（PostgreSQL）  
3. 调用 AI 生成中文分析（DeepSeek）  
4. 网站页面读取数据库并展示  
5. 用户在浏览器访问并交互

一句话记忆：

**Next.js 负责页面与 API，Supabase 负责数据，脚本负责进数，AI 负责分析，Vercel 负责上线。**

## 3. 从“用户看到页面”倒推技术链路

### 3.1 用户访问 `/`（Landing 首页）

文件链路：

1. `src/app/page.tsx`：路由入口  
2. `src/components/home/LandingPageUI.tsx`：首页主组件  
3. `src/app/layout.tsx`：全站根布局（字体、SEO、全局样式）

### 3.2 用户访问 `/home`（有真实数据）

文件链路：

1. `src/app/home/page.tsx`：服务端页面，负责请求数据  
2. `src/app/home/home-data.ts`：查询数据库 + 行情服务  
3. `src/app/home/HomePageClient.tsx`：客户端渲染和交互

这是典型的 “**服务端取数 + 客户端交互**” 模式。

### 3.3 用户访问 API（例如 `/api/companies`）

文件链路：

1. `src/app/api/companies/route.ts`：API 入口  
2. `src/lib/supabase.ts`：数据库客户端  
3. 返回 `NextResponse.json(...)` 给浏览器或前端组件

## 5. 环境变量（你必须懂）

示例在：`earnlytics-web/.env.example`

核心变量：

1. `NEXT_PUBLIC_SUPABASE_URL`：Supabase 地址  
2. `NEXT_PUBLIC_SUPABASE_ANON_KEY`：前端可用 key  
3. `SUPABASE_SERVICE_ROLE_KEY`：服务端高权限 key（不能泄露到前端）  
4. `FMP_API_KEY`：财报数据源  
5. `DEEPSEEK_API_KEY`：AI 分析

记忆规则：

1. `NEXT_PUBLIC_` 开头：可能暴露到前端  
2. 无 `NEXT_PUBLIC_`：仅服务端可用  
3. `SERVICE_ROLE_KEY` 只允许在服务端/API/脚本使用

## 7. TypeScript 新手入门（项目实战版）

你可以把 TypeScript 理解成：**带类型约束的 JavaScript**。

### 7.1 基础类型

```ts
const name: string = "Apple";
const price: number = 229.87;
const isHot: boolean = true;
```

### 7.2 对象类型

```ts
type Company = {
  id: number;
  symbol: string;
  name: string;
  sector: string | null;
};
```

对应本项目真实文件：`src/types/database.ts`

### 7.3 可选字段 `?`

```ts
type User = {
  id: string;
  nickname?: string;
};
```

`nickname?` 表示“可以没有这个字段”。

### 7.4 联合类型 `|`

```ts
type Sentiment = "positive" | "neutral" | "negative";
```

表示值只能三选一，项目 AI 情绪字段就是这种写法。

### 7.5 函数类型

```ts
function add(a: number, b: number): number {
  return a + b;
}
```

参数类型在前，返回类型在后。

### 7.6 `interface` vs `type`

在这个项目里二者都可用。

1. `type`：更灵活，常用于组合类型  
2. `interface`：常用于对象结构，支持声明合并

新手建议：先统一用 `type`，够用了。

### 7.7 数组与泛型

```ts
const symbols: string[] = ["AAPL", "MSFT"];
const ids: Array<number> = [1, 2, 3];
```

### 7.8 空值处理（很重要）

项目里经常有 `null` 数据（比如某些财务指标缺失）：

```ts
function formatRevenue(value: number | null): string {
  if (value === null) return "N/A";
  return `$${value.toLocaleString()}`;
}
```

### 7.9 可选链 `?.` 与空值合并 `??`

```ts
const name = company?.name ?? "Unknown";
```

意思：

1. `company` 为空时，不报错  
2. 最终拿不到值时，用 `"Unknown"`

### 7.10 异步语法 `async/await`

```ts
async function getCompanies() {
  const res = await fetch("/api/companies");
  return res.json();
}
```

项目里 API、数据库、AI 调用全部是异步。

### 7.11 为什么还要 Zod

TypeScript 只在“编译时”检查类型。  
用户请求进来时仍可能是脏数据。

所以项目在 `src/lib/ai.ts` 里用 `zod` 做运行时校验：

1. TS 保证“你写代码时”类型正确  
2. Zod 保证“真实输入数据”结构正确

## 8. TSX / React 语法（你在页面里最常见）

### 8.1 组件就是函数

```tsx
export default function Page() {
  return <div>Hello</div>;
}
```

### 8.2 Props 是输入参数

```tsx
type CardProps = { title: string };

function Card({ title }: CardProps) {
  return <h2>{title}</h2>;
}
```

### 8.3 `use client` 的意义

Next.js App Router 默认是“服务端组件”。  
只要你要用以下能力，就必须在文件顶部加 `"use client"`：

1. `useState`  
2. `useEffect`  
3. 浏览器事件（点击、输入）

项目示例：`src/app/home/HomePageClient.tsx`

### 8.4 动态导入 `dynamic(...)`

项目使用 `dynamic` 降低首屏体积（例如 `HomePageClient.tsx` 里延迟加载多个组件）。

## 9. Next.js App Router 你必须知道的 6 件事

1. `src/app/page.tsx` 对应 `/`  
2. `src/app/companies/page.tsx` 对应 `/companies`

4. `[symbol]` 是动态路由（如 `/api/stock-price/AAPL`）  
5. `layout.tsx` 是布局，包裹子页面  
6. `metadata` 用于 SEO 标题和描述

## 10. 文件之间如何协作（两条完整案例）

### 10.1 案例 A：`/home` 页面数据渲染

```text
浏览器 GET /home
  -> src/app/home/page.tsx
     -> fetchHomePageData() in src/app/home/home-data.ts
        -> supabase 查询 earnings/companies/ai_analyses
        -> getLatestStockPrices() in src/lib/stock-price-service.ts
     -> 返回 initialData 给 HomePageClient
  -> src/app/home/HomePageClient.tsx 渲染页面
```

### 10.2 案例 B：`/api/stock-price/[symbol]`

```text
浏览器 GET /api/stock-price/AAPL
  -> src/app/api/stock-price/[symbol]/route.ts
     -> getLatestStockPrice() 先读缓存
     -> 过期则 refreshStockPrice() 拉取实时数据
     -> 失败时回退旧缓存或开发 mock
  -> NextResponse.json 返回给前端
```

## 附录 A：JavaScript 零基础预备（读 TypeScript 前先过这一章）

> TypeScript = JavaScript + 类型系统。  
> 如果 JS 基础不稳，看 TS 会很痛苦。这个附录只讲你在本项目最常见的 JS 语法。

### A.1 变量：`const`、`let`

```js
const siteName = "Earnlytics"; // 不会重新赋值
let count = 0;                 // 可能变化
count = count + 1;
```

规则：

1. 默认用 `const`  
2. 变量会变化时才用 `let`  
3. 不建议用 `var`（老语法）

### A.2 字符串模板

```js
const symbol = "AAPL";
const text = `当前股票: ${symbol}`;
```

项目中常用于日志和提示文案拼接。

### A.3 条件判断

```js
if (count > 0) {
  console.log("有数据");
} else {
  console.log("无数据");
}
```

三元表达式：

```js
const label = count > 0 ? "有数据" : "无数据";
```

### A.4 数组常用方法（最重要）

```js
const symbols = ["AAPL", "MSFT", "NVDA"];

const upper = symbols.map((s) => s.toUpperCase());         // 映射
const filtered = symbols.filter((s) => s.startsWith("M")); // 过滤
const found = symbols.find((s) => s === "NVDA");           // 查找
```

项目里 `map/filter/find` 非常多，特别在页面渲染和数据转换时。

### A.5 对象与解构

```js
const company = { symbol: "AAPL", name: "Apple", sector: "消费电子" };
const { symbol, name } = company; // 解构
```

在 React 组件里最常见：

```tsx
function Card({ title, value }) {
  return <div>{title}: {value}</div>;
}
```

### A.6 展开语法 `...`

```js
const a = { x: 1, y: 2 };
const b = { ...a, y: 99 }; // 复制并覆盖
```

数组：

```js
const arr1 = [1, 2];
const arr2 = [...arr1, 3, 4];
```

项目里常用于“基于旧数据生成新数据”（不可变更新）。

### A.7 函数写法（两种都要认得）

```js
function add(a, b) {
  return a + b;
}

const add2 = (a, b) => {
  return a + b;
};
```

箭头函数在项目中更常见，尤其在 `map/filter` 和 React 组件回调里。

### A.8 模块系统：`import` / `export`

导出：

```js
export function formatCurrency(v) {
  return `$${v}`;
}
```

导入：

```js
import { formatCurrency } from "@/lib/utils";
```

你在项目中看到的文件协作，本质就是模块导入导出。

### A.9 异步：`Promise` 与 `async/await`

```js
async function loadCompanies() {
  const res = await fetch("/api/companies");
  const data = await res.json();
  return data;
}
```

理解要点：

1. `await` 只能在 `async` 函数里用  
2. `fetch`、数据库查询、AI 调用都是异步  
3. 异步代码必须考虑失败情况

### A.10 错误处理：`try/catch`

```js
try {
  const data = await loadCompanies();
  console.log(data);
} catch (error) {
  console.error("加载失败", error);
}
```

项目 API 中几乎都用了这个模式。

### A.11 空值安全（为什么经常看到 `?.` 和 `??`）

```js
const name = user?.profile?.name ?? "Unknown";
```

含义：

1. `?.`：前面为空就短路，不报错  
2. `??`：左边是 `null/undefined` 时用右边默认值

这对财务数据场景很关键，因为很多字段可能缺失。

### A.12 JS 到 TS 的升级关系（你这样记）

你先写 JS：

```js
function formatPrice(value) {
  return `$${value}`;
}
```

再升级为 TS：

```ts
function formatPrice(value: number): string {
  return `$${value}`;
}
```

区别就是加了类型约束，其他逻辑基本不变。

### A.13 结合本项目的“语法识别地图”

1. JS 数组处理：`src/app/home/home-data.ts`  
2. 异步请求：`src/app/api/**/route.ts`  
3. 解构 + 回调：`src/app/home/HomePageClient.tsx`  
4. 错误处理：`src/app/api/stock-price/[symbol]/route.ts`  
5. 导入导出：几乎所有 `src/**` 文件
