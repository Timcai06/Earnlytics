# 文件协作关系图

## 分层图

```text
[用户浏览器]
  -> [页面层] src/app/**/page.tsx
  -> [组件层] src/components/**
  -> [接口层] src/app/api/**/route.ts
  -> [服务层] src/lib/**
  -> [数据层] Supabase / 外部 API
```

## 你写功能时的工作顺序

1. 先确定页面入口在 `src/app` 哪个路由
2. 再确认要复用还是新增组件
3. 再确认是否需要 API
4. 需要数据库时在 `lib` 封装查询
5. 最后把返回数据映射成页面展示

## 一条实战路径（公司列表）

1. 页面：`src/app/companies/page.tsx`
2. 客户端交互：`src/app/companies/CompaniesList.tsx`
3. 数据来源：`companies` 表
4. 相关 API：`/api/companies`
